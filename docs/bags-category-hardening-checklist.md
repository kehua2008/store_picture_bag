# 箱包分站品类深改验收清单

本清单用于每次复刻、改版或上线前反查箱包站是否仍像一个“箱包电商视觉专家”，而不是服装站或鞋站的改名版本。

## 用户可见层

- 首页、工作台、账户、充值、生成记录、后台导航都使用箱包站身份。
- 上传提示围绕包款正面、侧面、背面、包身、底部、五金、拉链、肩带、内部结构、容量、持包参考图。
- 类目、模特、风格、套图模板、视频入口没有旧服装或鞋站营销表达。
- 普通用户看不到 prompt、negative prompt、seed、steps、denoise 等 AI 参数。

## 箱包生成规则

- 商品身份锁必须覆盖包型轮廓、包身比例、手柄、肩带、拉链路径、扣具、锁扣、五金、口袋布局、包底、轮组/拉杆、材质纹理、走线、边油、Logo-like 标记和印花。
- 风格参考只能影响背景、光线、镜头、构图、调色、修图质感和商业氛围，不能污染包款结构、颜色、Logo-like 标记、材质或五金。
- 白底、棚拍、场景、详情、海报、视频各角色都要有不同任务，避免 9 张详情图只是换背景。
- 平台规则和箱包规则同时生效：既能上架，又能让买家看清包款、容量和做工。

## 详情套图

- 基础转化版：白底、多色 SKU、持包场景、包型卖点、包身细节、五金工艺、尺寸容量、品质保养收尾。
- 通勤质感版：强化手提、肩背、斜挎、半身比例、皮革/帆布/尼龙质感和高级通勤氛围。
- 旅行容量版：强化旅行场景、完整箱包、容量分区、包身材质、五金拉链、轮组/拉杆、尺寸容量和耐用感。

## 视频语言

- 一句话生成和参考视频生成都按箱包镜头组织：包款亮相、45 度完整展示、包身/五金/拉链特写、容量或携带演示、通勤/旅行/户外场景、结尾定格。
- 参考视频只继承节奏、镜头和动作关系，不复制第三方品牌、人物身份、字幕、水印、Logo 或不相关商品结构。

## 隔离和回归

- Cookie、数据目录、上传目录、端口、进程名、清理脚本、健康检查都使用箱包站配置。
- 历史 `shoe_*` 风格 id 必须兼容映射到新箱包风格，不能导致旧草稿或已有记录报错。
- 上线前至少运行 `npm run typecheck`、相关领域单测和 `npm run verify:isolation`。
