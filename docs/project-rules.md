# 箱包电商 AI 生图生视频系统项目规则

本文件记录箱包站的硬性规则。任何新增功能不能违背这些规则；员工端不能暴露 prompt 或 AI 专业参数。旧服装 PRD 只作为底座能力来源，不作为箱包品类表达依据。

## 1. 系统定位

- 企业内部专用、零提示词、全固化模板、傻瓜式一键出图。
- 员工只允许点选、上传、生成、下载，不允许编辑 prompt、强度、步数、种子等 AI 参数。
- 生图、生视频规则、平台规范、箱包标准、负面词、风格模板全部后台内置。
- 以唯品会为核心，覆盖淘宝、拼多多、京东、抖音电商、得物。
- 支持单款生成和批量铺货队列。

## 2. 员工端必须包含

- 素材上传：支持包款白底图、正侧背角度图、包身/底部/五金/拉链/肩带细节图、持包图、容量或内部结构图；支持批量多张上传。
- 平台选择：唯品会、淘宝、京东、抖音电商、得物、拼多多。
- 箱包类目：女包、男包、双肩包、旅行箱、托特包、斜挎包、腋下包、公文包、电脑包、童包、小皮具、收纳/化妆包、户外机能包等。
- 图片类型：在平台规格下选择白底主图、持包/背包场景图、棚拍主图、侧背补充图、包身材质细节、五金/拉链/肩带结构、尺寸容量说明、多色 SKU、详情页海报和短视频封面。
- 风格选择：静奢皮具、城市通勤、高街潮流、韩系松弛、新中式雅致、甜酷多巴胺、运动机能、旅行场景、极简高级等箱包可理解风格。
- 尺寸规格：平台选择后自动匹配默认尺寸，并允许切换唯品会主图、淘宝 800x800、京东主图/详情/活动图、抖音方形/竖版、得物待校准质感图、拼多多规范尺寸、详情页长图；自定义规格必须保留目标宽高并映射到云雾支持尺寸。
- 模特选择：员工先选择“需要模特 / 不需要模特”；不需要模特时后端使用 `product_only`；需要模特时可选女包持包、男包通勤、儿童包、无露脸持包、真人背包/斜挎、手提/肩带局部、半身搭配持包，并可上传本地内存版自定义持包参考照。
- 出图数量：主图套图、详情基础转化版、通勤质感版、旅行容量版、自定义数量。
- 队列与结果：支持批量队列、进度展示、单图下载、批量打包下载、失败重跑、生成记录留存。

## 3. 前端展示规则

- 前端展示当前选择对应的样例预览，给员工建立初印象。
- 前端不得展示正向 prompt、负向 prompt、合规词库、反错误词库或模型参数。
- 用户端不展示后台导入的 prompt、prompt variants、negative prompt 或风格规则，只展示风格名称与面向用户的视觉信息。
- 平台合规规则可以影响后端 prompt 和后处理，但不在员工端直接展示为“禁止文字贴片”等规则清单。
- 前端只能展示平台规格、图片类型和样例预览，不展示后端拼接出的 prompt 或负面词。

## 4. 后台与模型规则

- 管理员端必须支持提示词模板管理、平台规范参数管理、箱包类目参数绑定、批量任务记录、失败重跑、用户权限。
- 候选图片不等于首页经典风格。候选批次必须经过 Codex Skill 离线分析、分析结果导入、管理员审核和发布，才能作为正式风格板进入首页。
- `manifest.json` 只是在后台导出分析包时生成的线上到本地同步清单，不是线上固定文件。
- 普通员工不可改参数；管理员可配置模板、上传 Lora、配置规则、查看日志。
- 说明书要求优先私有化部署，素材、训练数据、生图数据本地留存。
- 说明书要求 Flux/SDXL、ControlNet、Lora、SR 等能力；当前云雾 Image API MVP 只能作为过渡方案，不能视为最终完整交付。
- 后端必须内置全平台合规词库、箱包专属固定提示词库、反错误词库。

## 5. 质量与验收标准

- 包型不走样，包身比例、手柄、肩带、拉链路径、五金、口袋、包底、轮组和拉杆结构稳定。
- 皮革、帆布、尼龙、绒面、编织等材质纹理真实，不糊、不油、不 AI 塑料感。
- 光影均匀，符合电商棚拍标准。
- 背景干净、无杂物。
- 持包、肩背、斜挎、双肩背、拉杆行走、手部开合等动作自然，人体比例和包款尺度协调。
- 调试完成后可用率目标不低于 85%。
- 主图和详情图可批量生成，图片清晰度达到电商上架标准。

## 6. 当前实现状态

- 已实现：员工端上传、六平台选择、细分类目选择、平台规格内嵌图片类型选择、风格样例选择、规格选择、模特模式选择、本地内存自定义模特库、样例预览、队列状态、单图下载、云雾图像编辑调用。
- 已实现：前端隐藏 prompt，后端继续内置 prompt。
- 已实现：新上传素材会替换旧素材并刷新中间预览。
- 待实现：6 张/9 张套餐真实拆分队列、自定义数量、ZIP 打包、失败重跑、生成记录留存。
- 待实现：管理员后台、模板管理、平台规范管理、权限系统。
- 待实现：私有化 Flux/SDXL、ControlNet、Lora、SR、纯白底后处理、详情页全套自动拼接。

## 8. 文档治理规则

- 改代码必须同步对应细粒度文档，尤其是 API、领域模型、后台流程、用户端生成逻辑、导出/导入 schema 和 Codex Skill 约定。
- 风格库相关改动必须检查 `docs/style-library/*` 与 `docs/frontend/classic-style-selection.md`。
- 文档同步细则见 `docs/rules/documentation-sync.md`。

## 7. 平台规格来源记录

- 唯品会：商品资料规范 PDF（2021）明确主图数量与浅色背景要求；MVP 输出尺寸沿用项目现有 1340x1785、1200x1200 规则。
- 抖音：Canva 抖音尺寸说明与公开电商尺寸汇总，覆盖 800x800、750x1000、1080x1920、750x550、详情宽 750 等用法。
- 淘宝/京东/拼多多：公开电商尺寸汇总资料，覆盖主图、长图、详情宽度与活动图常见尺寸。
- 得物：未找到可靠公开官方尺寸明细，当前只加入质感主图、SKU/透明图、细节图、场景图可编辑预设，并标记为“待商家后台校准”。
